### 问题 2：img_crop.py 中 crop_transparent 坐标变量名与实际含义不匹配

**文件**：`filekits/image/img_crop.py`
**行号**：第 17-20 行
**级别**：🔴 严重bug

**问题描述**：
`np.argwhere()` 返回的是 `(row, col)` 即 `(y, x)` 格式的坐标数组，但代码将 `coords.min(axis=0)` 赋值给 `x0, y0`，`coords.max(axis=0)` 赋值给 `x1, y1`。这意味着实际赋值顺序是 `x0 = min(y)`, `y0 = min(x)`, `x1 = max(y)`, `y1 = max(x)`，与变量名暗示的含义完全相反，可能导致裁剪区域错误。

**代码片段**：
```python
coords = np.argwhere(img[..., 3])
x0, y0 = coords.min(axis=0)   # x0 = min(y), y0 = min(x)，命名与含义相反
x1, y1 = coords.max(axis=0) + 1  # x1 = max(y), y1 = max(x)
cropped_img = img[x0:x1, y0:y1]  # 实际裁剪范围与预期不符
```

**修复建议**：
```python
coords = np.argwhere(img[..., 3])
y0, x0 = coords.min(axis=0)   # row = y, col = x
y1, x1 = coords.max(axis=0) + 1
cropped_img = img[y0:y1, x0:x1]  # 正确对应 row(y) 和 col(x)
```

**代码片段**：
```python
def remove_keys(data: dict, keys_to_remove: list) -> dict:
    for key in keys_to_remove:
        if key in data:
            del data[key]  # 原地修改入参
    return data
```

**修复建议**：
```python
def remove_keys(data: dict, keys_to_remove: list) -> dict:
    return {k: v for k, v in data.items() if k not in keys_to_remove}
```
