### 问题 1：clear_folder 存在并发竞态条件（忽略）

**文件**：`filekits/base_io/folder.py`
**行号**：第 19–23 行
**级别**：🔴 严重bug

**问题描述**：
`clear_folder` 函数先将目录删除（`shutil.rmtree`），随后立即重建同名目录（`os.mkdir`）。在并发场景下，若其他代码逻辑正在访问该目录，可能导致文件操作失败或目录被意外清空。此外，目录删除后重建之间存在时间窗口，多线程/多进程程序中可能出现 `FileNotFoundError`。

**代码片段**：
```python
def clear_folder( folder_path: StrPath ) :
    # 检测文件夹是否存在，如果不存在就创建
    if not os.path.exists( folder_path ) :
        os.makedirs( folder_path )
    shutil.rmtree( folder_path )   # ← 删除整个目录树
    os.mkdir( folder_path )         # ← 立即重建同名空目录
```

**修复建议**：
改用遍历清空目录内容而非删除目录本身，或在整个操作前后加锁保护。以下为安全清空方案：

```python
def clear_folder( folder_path: StrPath ) :
    if not os.path.exists( folder_path ) :
        os.makedirs( folder_path )
        return
    for item in os.listdir( folder_path ) :
        item_path = os.path.join( folder_path, item )
        if os.path.isdir( item_path ) :
            shutil.rmtree( item_path )
        else :
            os.remove( item_path )
```
